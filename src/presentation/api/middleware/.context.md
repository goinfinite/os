API-specific middleware for request/response processing, authentication, and cross-cutting concerns specific to REST API.

## Summary

- Authentication middleware for Bearer tokens and API keys
- Default header injection
- Trailing slash normalization
- Read-only mode enforcement
- Database service injection into request context

## Constraints

- MUST verify JWT or API key before allowing access to protected endpoints
- MUST NOT require authentication for /api/v1/auth/login or /api/v1/setup/* endpoints
