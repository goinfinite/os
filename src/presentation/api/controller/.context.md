HTTP request handlers for REST API. Each controller handles one domain (accounts, databases, mappings, etc.) and delegates to use cases for business logic.

## Summary

- authentication.go — Login endpoint
- account.go — Account CRUD and key management
- database.go — Database operations (create, read, delete, user management)
- marketplace.go — Marketplace catalog and installation
- cron.go — Scheduled task management
- virtualHost.go — Virtual host management
- runtime.go — Runtime configuration and management
- ssl.go — SSL certificate management
- services.go — Service deployment and lifecycle
- files.go — File upload, download, management
- setup.go — Initial setup wizard
- scheduledTask.go — Scheduled task execution
- o11y.go — System overview and monitoring

## Constraints

- MUST bind JSON request bodies to DTOs, validate, then call use cases
- MUST format DTO responses as JSON
- MUST handle domain errors and return appropriate HTTP status codes

## Guidance

- SHOULD keep controllers under 100 lines by delegating to use cases
- SHOULD use meaningful HTTP methods (GET/POST/PUT/DELETE)
