REST API layer providing programmatic access to all Infinite OS features. Uses Echo web framework with Swagger/OpenAPI documentation auto-generated from code annotations.

## Summary

- api.go — API initialization, middleware setup, request timeout configuration
- router.go — Route registration for all API endpoints (v1/)
- controller/ — HTTP request handlers for each feature domain
- docs/ — Auto-generated Swagger/OpenAPI documentation
- helper/ — API utilities (response formatting, error handling)
- middleware/ — API-specific middleware (auth, headers, read-only mode)

## Constraints

- MUST generate valid Swagger documentation automatically
- MUST support Bearer tokens (JWT) and API keys in Authorization header
- MUST return consistent error responses
- MUST enforce authentication for all endpoints except /api/v1/auth/login and /api/v1/setup/*

## Guidance

- SHOULD version API endpoints (currently v1/)
- SHOULD use HTTP status codes correctly
- SHOULD document all endpoints with Swagger annotations
