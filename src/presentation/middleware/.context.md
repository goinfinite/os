Cross-cutting HTTP middleware shared across the API and UI surfaces. Middleware in this directory runs inside the global Echo middleware chain registered in `http.go` and applies to every incoming request before route dispatch. Adjacent middleware (panic recovery, default headers) live in tk (`ApiPanicHandler`) and `presentation/api/middleware` respectively — those packages, not this one, own those concerns.

## Summary

- baseHref.go — Sets `rootBasePath`, `apiBasePath`, `uiBasePath`, and `baseHref` on the Echo context; supports `X-Base-Href` header override for reverse-proxy deployments

## Constraints

- MUST NOT depend on API or UI specific packages; this layer is shared
- MUST set `rootBasePath`, `apiBasePath`, `uiBasePath`, and `baseHref` on the Echo context before any handler or downstream middleware reads them
- MUST execute before any handler that reads `baseHref` for redirect/asset paths

## Guidance

- SHOULD register after global panic recovery but before route-level authentication
- SHOULD prefer additions to this package over duplicating context-key wiring inside `presentation/api/middleware` or `presentation/ui/middleware`
