User-facing interfaces for Infinite OS: REST API, CLI, and web UI. This layer handles routing, request binding, response formatting, authentication, and presents domain logic to end users. The http.go file bootstraps the web server with all three interfaces.

## Summary

- http.go — HTTP server initialization, SSL setup, and banner output
- api/ — REST API with Swagger/OpenAPI documentation
- cli/ — Command-line interface using Cobra
- ui/ — Web dashboard with embedded assets
- middleware/ — Cross-cutting concerns (auth, panic handling, headers)
- shared/ — Utilities shared across presentation interfaces
- liaison/ — Inter-process communication handlers

## Constraints

- MUST NOT contain business logic; only request/response handling
- MUST authenticate all requests except setup wizard
- MUST validate all inputs before passing to domain/
- MUST use DTOs for request/response payloads

## Guidance

- SHOULD keep controllers thin; delegate to use cases
- SHOULD separate API, CLI, and UI concerns
