CLI-specific middleware for panic handling, root access checking, environment validation, primary virtual host synchronization, and logging setup.

## Summary

- PanicHandler — Catches and recovers from panics
- PreventRootless — Ensures CLI runs with elevated privileges
- CheckEnvs — Validates required environment variables
- PrimaryVirtualHostSynchronizer — Syncs PRIMARY_VHOST across env, nginx conf, and DB on every boot
- LogHandler — Configures logging for CLI operations

## Constraints

- MUST prevent execution without root privileges
- MUST validate all required environment variables at startup
- MUST sync primary virtual host state after DB init and before route registration
