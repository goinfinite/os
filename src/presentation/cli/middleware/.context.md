CLI-specific middleware for environment validation and root privilege enforcement. These run during CLI bootstrap before any command dispatch.

## Summary

- checkEnvs.go — Validates required environment variables (ACCOUNT_API_KEY_SECRET, JWT_SECRET) and auto-fills missing ones
- preventRootless.go — Ensures CLI runs as root; exits early in DEV_MODE

## Constraints

- MUST prevent execution without root privileges (unless in DEV_MODE)
- MUST validate all required environment variables at startup
- MUST NOT open or query any internal database during env validation (file/env/OS-level only)

## Guidance

- SHOULD keep middleware pure: validate state, exit on fatal misconfiguration, do not silently fix missing config
- SHOULD treat `DEV_MODE` as a developer-only escape hatch for environment-validation and root-privilege checks
- SHOULD NOT introduce persistent side effects (no DB writes, no file mutations) from this layer — defer side effects to the use-case that needs them
