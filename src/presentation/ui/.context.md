Web dashboard UI rendered server-side using Go template components. Provides a graphical interface for managing applications, databases, accounts, SSL, and mappings. Assets (CSS, JS) are embedded in the binary.

## Summary

- ui.go — UI initialization and asset embedding
- router.go — Route registration for all UI pages
- assets/ — Static files (CSS, JavaScript, images) embedded in binary
- component/ — Reusable template components (forms, tables, modals, etc.)
- layout/ — Page layout templates (main, login, setup, sidebar, footer)
- presenter/ — Page-level template rendering for each feature
- middleware/ — UI-specific middleware (authentication, asset serving)
- helper/ — UI utilities (template helpers, formatters)

## Constraints

- MUST serve all assets from embedded FS, not from disk
- MUST authenticate users before showing any non-login pages
- MUST regenerate templates on every request in dev mode

## Guidance

- SHOULD use reusable components to reduce duplication
- SHOULD keep presenter files focused on their specific page
