Page presenters rendering feature-specific UI pages. Each presenter prepares data and renders templates for a specific feature area.

## Summary

- accounts/ — Account management pages
- crons/ — Scheduled task management pages
- databases/ — Database management pages
- fileManager/ — File browser and management pages
- login/ — Login page
- mappings/ — Mapping configuration pages
- marketplace/ — Application marketplace pages
- overview/ — System overview/dashboard page
- runtimes/ — Runtime configuration pages
- setup/ — Initial setup wizard pages
- ssls/ — SSL certificate management pages
- helper/ — Shared presenter utilities
- footer/ — Footer presenter
- And more feature areas...

## Constraints

- MUST authenticate user before rendering any page except login/setup
- MUST pass data through DTOs to templates, never raw entities

## Guidance

- SHOULD keep presenters focused on a single feature or page
- SHOULD delegate business logic to use cases
