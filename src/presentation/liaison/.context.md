Service liaison wrappers that instantiate use cases and repositories for CLI and local operations. Each file corresponds to a domain area and wires together the infrastructure repos, database services, and domain use cases.

## Summary

- account.go — AccountLiaison: manages account CRUD and password changes
- authentication.go — AuthenticationLiaison: handles login, logout, and token refresh
- cron.go — CronLiaison: manages cron job CRUD operations
- database.go — DatabaseLiaison: manages database CRUD and user privileges
- marketplace.go — MarketplaceLiaison: manages marketplace item installs, upgrades, and removals
- o11y.go — O11yLiaison: provides observability overview (CPU, RAM, disk metrics)
- runtime.go — RuntimeLiaison: manages PHP and runtime environment configuration
- scheduledTask.go — ScheduledTaskLiaison: manages scheduled task CRUD and execution
- services.go — ServicesLiaison: manages service installations, start/stop, and configuration
- ssl.go — SslLiaison: manages SSL certificate and pair CRUD, including Let's Encrypt
- virtualHost.go — VirtualHostLiaison: manages virtual host and mapping CRUD
