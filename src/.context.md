The source code for Infinite OS, a self-hosted application platform built in Go. This follows a clean architecture pattern with distinct layers: domain (business logic), infra (infrastructure concerns), and presentation (API, CLI, UI).

## Summary

- domain/ — Core business logic: value objects, entities, use cases, DTOs, and repository interfaces
- infra/ — Infrastructure layer: database services, file operations, authentication, SSL, services, and runtime management
- presentation/ — User-facing interfaces: REST API, CLI, web UI, middleware, and routing
- devUtils/ — Development utilities for testing and debugging

## Constraints

- MUST follow clean architecture principles: domain logic depends on nothing else; infra/presentation depend on domain
- MUST NOT place infrastructure concerns in domain logic
- Database operations MUST go through internalDatabase services (persistent, transient, trail)

## Guidance

- SHOULD keep value objects and entities in domain/ with minimal dependencies
- SHOULD implement repository patterns in infra/ but define interfaces in domain/
- SHOULD use dependency injection for service initialization
