Infrastructure layer for virtual host and URL mapping management. Provides repository implementations for reading and writing NGINX vhost configurations and URL mappings using Go templates, plus helper functions for hostname resolution and config validation.

## Summary

- virtualHostCmdRepo.go — Command repository for creating and deleting virtual host configs (template-based NGINX config generation)
- virtualHostQueryRepo.go — Query repository for reading virtual host configurations from disk
- virtualHostHelpers.go — Helper utilities: reads primary vhost hostname from NGINX conf, validates config syntax, triggers NGINX reload
- mappingCmdRepo.go — Command repository for creating and deleting URL mapping configs within vhost files
- mappingQueryRepo.go — Query repository for reading URL mappings from persistent database

## Constraints

- MUST generate valid NGINX configuration
- MUST reload NGINX after configuration changes
- MUST NOT depend on infra/runtime (orchestration belongs in domain use cases)
