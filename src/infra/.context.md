Infrastructure layer providing technical implementations for persistence, external services, SSL, file operations, and runtime management. This layer implements domain repository interfaces and handles all dependencies on external systems.

## Summary

- account/ — Account repository implementations
- activityRecord/ — Activity logging persistence
- auth/ — Authentication and token handling
- cron/ — Scheduled task management
- database/ — Database service implementations (PostgreSQL, MySQL, etc.)
- envs/ — Environment variable access and configuration
- files/ — File system operations
- helper/ — Infrastructure utilities (SSL generation, file operations, etc.)
- internalDatabase/ — SQLite database services for persistent, transient, and trail data
- marketplace/ — Application marketplace logic
- o11y/ — Observability and system overview
- runtime/ — Runtime environment setup (PHP, Node, etc.)
- scheduledTask/ — Task scheduler implementation
- services/ — Service deployment and management
- ssl/ — SSL certificate handling
- vhost/ — Virtual host configuration
- webServer/ — Web server setup and configuration

## Constraints

- MUST NOT leak infrastructure details to domain or presentation layers
- MUST implement domain repository interfaces
- Database queries MUST use the appropriate service (persistent, transient, or trail)

## Guidance

- SHOULD encapsulate technology-specific code (SQL, external APIs) within infra/
- SHOULD use dependency injection for external service clients
