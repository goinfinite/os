SQLite database services managing three separate databases for different concerns: persistent (application state), transient (runtime data, cache), and trail (audit logs). All application data is stored through these services.

## Summary

- persistentDatabaseService.go — Manages persistent application state (accounts, mappings, services, etc.)
- transientDatabaseService.go — Manages temporary/cache data and runtime state
- trailDatabaseService.go — Manages audit logs and activity records
- model/ — Database schema models and table definitions
- repo/ — Repository implementations for data access
- helper/ — Database helper utilities

## Constraints

- MUST use only these three database services; no direct database access allowed
- Persistent service MUST preserve data across restarts
- Transient service data MAY be cleared on restart
- Trail service MUST maintain audit history indefinitely
- MUST use prepared statements and parameterized queries

## Guidance

- SHOULD keep database operations simple and focused
- SHOULD use model/ definitions for schema consistency
