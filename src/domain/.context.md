Core domain layer containing pure business logic independent of any infrastructure. This holds all entities, value objects, data transfer objects, use cases, and repository interfaces that define application behavior.

## Summary

- entity/ — Domain entities representing core business concepts
- valueObject/ — Type-safe, immutable value objects enforcing business rules
- valueObject/helper/ — Secure string hashing (all type conversion helpers were migrated to tk)
- dto/ — Data Transfer Objects for request/response payloads and inter-layer communication
- useCase/ — Business logic operations organized by feature
- repository/ — Interface definitions for data persistence (implementations in infra/)

## Constraints

- MUST NOT import from infra/ or presentation/
- Value objects MUST be immutable and validate invariants at construction
- MUST use DTOs for API/CLI input and output, never pass entities directly

## Guidance

- SHOULD keep use cases focused on single responsibilities
- SHOULD define repository interfaces here even if implementations live in infra/
