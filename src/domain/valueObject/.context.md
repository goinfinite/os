Immutable, type-safe value objects that represent domain concepts with validation. These enforce business rules at the type level, making invalid states unrepresentable. Examples: email addresses, file paths, access tokens, numeric identifiers.

## Summary

- helper/ — Cryptographic string hashing (SHA-3/256) for SSL certificate and pair IDs
- accessTokenStr.go, accessTokenType.go — Token value objects for authentication
- accountId.go, accountIdList.go — Account identifier value objects
- activityRecordCode.go, activityRecordId.go — Activity logging value objects
- Dozens more organized by domain concept (cron, database, file path, hostname, etc.)

## Constraints

- MUST be immutable after construction
- MUST validate invariants in constructor (New* functions)
- MUST have String() method for readable representation
- MUST NOT expose internal state directly; use accessor methods

## Guidance

- SHOULD use dedicated types rather than strings/ints for IDs and special values
- SHOULD keep validation logic inside the value object constructor
