Repository interfaces defining contracts for data access. These interfaces are implemented by the infra layer but declared here to keep domain logic decoupled from storage technology.

## Summary

- Interface definitions for persisting and querying domain entities
- Each repository handles one entity type (Account, Database, Cron, etc.)
- Concrete implementations exist in infra/

## Constraints

- MUST define methods that operate on domain entities, not raw data
- MUST NOT include storage-specific logic (SQL, schema details)
