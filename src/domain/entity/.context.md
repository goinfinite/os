Domain entities representing core business concepts and aggregates. These model the real-world objects that the system manages: accounts, databases, services, mappings, etc.

## Summary

- account.go — Account entity with identity and metadata
- database.go — Database entity representing a persistent data store
- Other entities for mappings, crons, services, and runtime configurations

## Constraints

- MUST encapsulate business rules and invariants
- MUST be serializable (for storage) but MUST NOT depend on specific storage technology
- Identity MUST be immutable once created

## Guidance

- SHOULD keep entities focused on a single responsibility
- SHOULD validate state transitions within the entity
