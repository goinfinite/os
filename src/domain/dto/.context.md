Data Transfer Objects that standardize request and response payloads across all presentation layers (API, CLI, UI). These are the contracts between user-facing interfaces and domain logic.

## Summary

- Various DTO files organized by feature (account, auth, database, cron, etc.)
- Each DTO encapsulates the fields needed for a specific API/CLI operation
- DTOs are used in both request binding and response marshaling

## Constraints

- MUST NOT contain business logic; DTOs are pure data containers
- MUST use JSON/YAML tags for API serialization
- MUST validate structural constraints at the presentation layer before mapping to DTOs

## Guidance

- SHOULD keep DTOs flat and simple for easy serialization
- SHOULD mirror the API response structure exactly
